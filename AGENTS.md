# Project Research Instructions

Follow the parent harness contract before touching a claim or conclusion.

- Start from `PROBLEM.md` and `DEFINITIONS.md`; preserve the exact statement.
- Keep claim and obligation JSON records valid. Run the harness checks from its
  repository after changes.
- Put source verification in `KNOWN_RESULTS.md`, computational provenance in
  `ARTIFACT_MANIFEST.md`, and failed approaches in `RESEARCH_LOG.md`.
- Use separate role reports. Do not overwrite another role's evidence.
- End a task in `RESULTS.md` with changed claims, evidence, commands, negative
  results, unresolved obligations, next actions, and a human-gate decision.
